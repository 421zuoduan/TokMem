# 周报-20260326

代码库与论文写的设置并不完全相同, 需要考虑自己完善他的代码, 使用自己的 setting. 目前做到的我认为的最好的 routing acc 只有 56.6%, 基本设置如下:

1. full vocab generation: 生成 memory token 时是从全词表生成, 而不只是从 memory token 对应词表中生成
2. prompt 采用 `instruction+query`: 训练和推理都会加任务描述, 推理阶段可以加一个只有 query 的实验; 也可以试试不加 `instruction`, 训练推理都是 `query only`, 这个还没试
3. max length = 1024: 论文与代码库有很多不同, 我自定义了
4. 词表 embedding 初始化: 固定 pretrained embedding 平均值
5. dataloader 设置 shuffle=false: 代码库和 issue 都说设置 false, 不过感觉设置 true 会更好? 需要再思考
6. task 最多用 700 个: 论文说是用了 1000 个样本, 但数据集筛选后最多只有 763 个任务满足 train500+eval10+test50 的样本数量要求

## 1. 论文、官方源码、当前实验的差异

1. memory token 的 routing 生成
    - full vocab generation: memory token 从全词表生成
    - first step generation: memory token 从 memory token 相关的词表中生成
2. dataloader shuffle: 论文是逐任务训练, 所以不 shuffle. 这是目前最怀疑的点, 使用 10task 时有不错的效果, 但是任务量上来时效果会下降很多, 怀疑改变 shuffle 会提高效果, 但与论文设置相悖. 需要继续思考合理性
3. prompt 形式: 论文和代码的训练和推理都是 `instruction+query`, 是不是可以加上一个推理时 `query-only` 的评估实验
